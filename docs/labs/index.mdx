# Kubernetes LLM Labs

The labs turn the guide into operator challenges. The docs remain free, indexable runbooks. The interactive version lives at [labs.k8sllm.online](https://labs.k8sllm.online), where you can type commands in a simulated terminal, inspect realistic Kubernetes output, unlock hints, reveal solutions, and track anonymous progress in your browser.

## Lab prerequisites

| Requirement | Why it matters |
| --- | --- |
| Kubernetes cluster access | You need namespace, workload, service, and log visibility. |
| GPU node pool for inference labs | LLM serving exercises depend on accelerator scheduling and runtime health. |
| Metrics stack | Prometheus, Grafana, OpenTelemetry, or equivalent telemetry makes validation concrete. |
| GitOps or manifest workflow | Labs are easier to review when every change is versioned. |
| Test prompts and evaluation cases | LLM workloads need behavior checks, not only pod readiness. |

## How to use the labs

1. Read the related architecture guide before applying manifests.
2. Open the matching interactive challenge at [labs.k8sllm.online/challenges](https://labs.k8sllm.online/challenges).
3. Run the challenge scenario in your own cluster or mock environment.
4. Paste command output into the guided checks and use hints only when blocked.
5. Trigger one failure drill and record which signal detected the failure first.

New labs should follow the [Content Review Checklist](/docs/content-review-checklist): objective, prerequisites, manifests or commands, validation signals, failure drills, and expected signals.

## Roadmap direction

| Version | Capability | Why it comes next |
| --- | --- | --- |
| V1 | Simulated terminal, guided checks, hints, local progress, and static docs | This is backend-free, searchable, and useful before hosted sandboxes exist. |
| V2 | Local sandbox kits with `kind`, `k3d`, Docker Compose, and downloadable manifests | This makes the labs reproducible without paying for hosted infrastructure. |
| V3 | Hosted browser labs | This needs auth, abuse protection, sandbox lifecycle management, and an infrastructure budget. |

## Recommended path

Start with [K8s LLM: Kubernetes LLM Platform Guide](/docs/k8s-llm), continue to [GPU Node Pool Kubernetes](/docs/llm-on-kubernetes/gpu-node-pools), run the [vLLM Inference Challenge](https://labs.k8sllm.online/challenges/vllm-inference), then move to [RAG on Kubernetes](/docs/llm-on-kubernetes/rag-on-kubernetes) and the [RAG Retrieval Challenge](https://labs.k8sllm.online/challenges/rag-retrieval).
