# Publish And Domain

This repository is configured for GitHub Pages first. The target public URL is:

```text
https://vtmhieu.github.io/k8sllm/
```

## Local run

```bash
npm install
npm run start
```

Open the local URL printed by Docusaurus.

## GitHub Pages

1. In repository settings, open **Pages**.
2. Set **Source** to **GitHub Actions**.
3. Push to `main`.
4. The `Build and Deploy` workflow builds the site and deploys the `build/` artifact.
5. Expected public URL:

```text
https://vtmhieu.github.io/k8sllm/
```

The current `docusaurus.config.js` GitHub Pages settings are:

```js
url: 'https://vtmhieu.github.io',
baseUrl: '/k8sllm/',
organizationName: 'vtmhieu',
projectName: 'k8sllm',
```

Keep `static/.nojekyll` so GitHub Pages does not process Docusaurus output with Jekyll.

## Custom domain

1. Choose the real domain, for example `kubernetes.example.com`.
2. Copy `static/CNAME.example` to `static/CNAME` and replace the placeholder with the real domain.
3. Update `docusaurus.config.js`:

```js
url: 'https://kubernetes.example.com',
baseUrl: '/',
```

4. Configure DNS:

```text
Type: CNAME
Name: kubernetes
Value: vtmhieu.github.io
```

5. In GitHub Pages settings, set the custom domain and enable HTTPS.

## Cloudflare Pages

Use this when you want simple custom domain handling and preview deployments.

```text
Build command: npm run build
Build output: build
Node version: 20 or newer
```

For Cloudflare Pages with a root custom domain, set:

```js
url: 'https://kubernetes.example.com',
baseUrl: '/',
```
