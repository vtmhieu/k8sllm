# Publish And Domain

This folder is intentionally self-contained so it can become a standalone Git repository later.

## Local run

```bash
npm install
npm run start
```

Open the local URL printed by Docusaurus.

## GitHub Pages

1. Create a GitHub repository named `k8sllm`.
2. Push this folder as the repository root.
3. In repository settings, enable Pages from GitHub Actions.
4. Update `docusaurus.config.js`:

```js
url: 'https://vtmhieu.github.io',
baseUrl: '/k8sllm/',
organizationName: 'vtmhieu',
projectName: 'k8sllm',
```

5. Keep `static/.nojekyll` so GitHub Pages does not process Docusaurus output with Jekyll.

## Custom domain

1. Choose the real domain, for example `kubernetes.example.com`.
2. Copy `static/CNAME.example` to `static/CNAME` and replace the placeholder with the real domain.
3. Update `docusaurus.config.js`:

```js
url: 'https://kubernetes.example.com',
baseUrl: '/',
```

4. Configure DNS:

```text
Type: CNAME
Name: kubernetes
Value: YOUR_GITHUB_USERNAME.github.io
```

5. In GitHub Pages settings, set the custom domain and enable HTTPS.

## Cloudflare Pages

Use this when you want simple custom domain handling and preview deployments.

```text
Build command: npm run build
Build output: build
Node version: 20 or newer
```

For Cloudflare Pages with a root custom domain, set:

```js
url: 'https://kubernetes.example.com',
baseUrl: '/',
```
