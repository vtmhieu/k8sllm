# Production Guides

These guides target the searches that usually come from a real platform problem: slow first token, pending GPU pods, serving framework decisions, RAG access failures, and launch readiness.

> Last reviewed: June 8, 2026. These pages are source-anchored and designed as entry points into the matching K8sLLM labs.

| Guide | Production problem | Matching lab |
| --- | --- | --- |
| [LLM Latency on Kubernetes](/docs/production-guides/llm-latency-on-kubernetes) | Pods are healthy, but users still wait for first token. | [vLLM inference challenge](https://labs.k8sllm.online/challenges/vllm-inference) |
| [vLLM Kubernetes Production Deployment](/docs/production-guides/vllm-kubernetes-production-deployment) | Runtime is deployed, but model readiness, cache, probes, and metrics are unclear. | [vLLM Kubernetes deployment lab](https://labs.k8sllm.online/challenges/vllm-kubernetes-deployment) |
| [GPU Node Pool Scheduling for LLM Inference](/docs/production-guides/gpu-node-pool-scheduling-llm-inference) | Expensive GPU capacity exists, but pods are pending or fragmented. | [GPU node pool scheduling lab](https://labs.k8sllm.online/challenges/gpu-node-pool-scheduling) |
| [KServe vs Ray Serve for LLM Platforms](/docs/production-guides/kserve-vs-ray-serve-llm-platforms) | The team is choosing a serving layer without agreeing on ownership. | [KServe vs Ray Serve decision lab](https://labs.k8sllm.online/challenges/kserve-vs-ray-serve-decision) |
| [RAG Tenant Isolation on Kubernetes](/docs/production-guides/rag-tenant-isolation-kubernetes) | Retrieval quality looks good, but tenant boundaries are not proven. | [RAG retrieval challenge](https://labs.k8sllm.online/challenges/rag-retrieval) |
