# Reference Architectures

These architectures are review blueprints. They do not replace organization-specific design, but they help platform teams avoid missing important boundaries.

## Architecture set

| Architecture | Use when |
| --- | --- |
| [Production Cluster](/docs/reference-architectures/production-cluster) | You need a baseline cluster design. |
| [Multi-Tenant Security](/docs/reference-architectures/multi-tenant-security) | Multiple teams or tenants share infrastructure. |
| [Observability Pipeline](/docs/reference-architectures/observability-pipeline) | You need standardized metrics, logs, traces, and events. |
| [GitOps Flow](/docs/reference-architectures/gitops-flow) | You need deployment flow and drift control. |
| [LLM Inference Stack](/docs/reference-architectures/llm-inference-stack) | You run model serving on GPU node pools. |
| [RAG Platform](/docs/reference-architectures/rag-platform) | You design production retrieval augmented generation. |
